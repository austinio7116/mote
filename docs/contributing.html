Contribute a Mote game

Gallery Engine docs GitHub

Built a game on the Mote engine and want it in the gallery? Here's the whole path — fork, build, add your game to the listing, and open a pull request for review. It takes about ten minutes once your game is finished.

Before you start: your game should already build and play in the emulator. If you're still writing it, see the README for the full engine + build setup. This page is only about submitting a finished game.

1 Fork & clone

Fork austinio7116/mote on GitHub (the Fork button, top-right), then clone your fork and make a branch:

git clone https://github.com/<you>/mote.git
cd mote
git checkout -b add-mygame

2 Add your game to games/

Each game is a self-contained folder under games/. Drop yours in as games/mygame/ with this shape:

games/mygame/
  ├─ src/game.c  — your game (+ any baked .h asset headers)
  ├─ assets/  — editable source art / audio / levels
  ├─ icon.png  — 60×60 launcher icon (required)
  └─ arcade_description.txt  — one-line blurb

Two macros in game.c are the source of truth for your listing — make sure they're set:

MOTE_GAME_META("My Game", "yourname");   // name + author
MOTE_GAME_VERSION("1.0.0");              // bump this every release
Credit honestly. Put your own name as the author. If you used an art or audio pack, credit it in arcade_description.txt or a comment.

3 Build the .mote for the gallery

Build the device binary and copy it into docs/games/ (this is the file players download):

# from the repo root
./tools/mote build games/mygame --device
cp games/mygame/build/MyGame.mote docs/games/MyGame.mote

The .mote filename is your game's name from MOTE_GAME_META with spaces removed (e.g. My Game → MyGame.mote). Run ./tools/mote build games/mygame once first to see the exact name it prints.

4 Add your entry to the listing

The gallery is driven by a GAMES array in docs/games.html. Open it, scroll to the end of that array, and add your game as the last entry (keep the trailing comma tidy):

{id:"mygame", file:"MyGame.mote", name:"My Game", by:"yourname", kb:64,
 guide:"mygame-guide.html",          // optional — omit if you have no guide page
 mp:1,                               // optional — include ONLY if it's 2-player
 tag:"One-line hook for the hero card",
 desc:"A paragraph describing the game — what you do, what's special, the modes."},
  • id — lowercase, no spaces (matches your folder name).
  • file — must match the .mote you copied into docs/games/.
  • kb — rough download size in KB (the build step prints the byte size; divide by 1024).
  • tag — the short hook shown on the card. desc — the longer blurb on the hero panel.
  • mp:1 — add this only for multiplayer games; it shows the 2P badge.
  • guide — only if you also add a docs/<id>-guide.html page. Otherwise leave it out.

5 Regenerate games.json

Don't hand-edit docs/games.json — it's generated. Run the tool, which reads your games.html entry and your .mote (pulling out the embedded version, ABI, size and checksum):

python3 tools/gen_gallery.py

You should see your game printed in the summary, and docs/games.json will now include it. This is the manifest the in-Studio and on-device galleries read, so it has to be regenerated whenever a .mote or listing changes.

6 Commit

Commit your game code, the built .mote, and the two generated/edited listing files together:

git add games/mygame \
        docs/games/MyGame.mote \
        docs/games.html \
        docs/games.json
git commit -m "Add My Game to the gallery"

If you added a guide page or gallery images, include those too (docs/mygame-guide.html, docs/img/gallery/mygame-*.png).

7 Push & open a pull request

git push origin add-mygame

Then on GitHub, open your fork and click Compare & pull request (or the Contribute button) targeting austinio7116/mote · main. In the PR description, a few lines help the review:

  • What the game is, in a sentence or two.
  • Confirm it builds and plays in the emulator (and on a device, if you tested one).
  • Credit any art/audio/code you didn't make yourself.

That's it — the PR will be reviewed for inclusion. Thanks for building on Mote!
